📖 독서노트 (my booklog)

[1] 개요

기획 배경

  • 기록은 직관적인 것이 중요하다고 생각합니다.
  • 독서는 지식과 영감을 얻을 수 있는 소중한 활동이지만, 그 내용을 기억하고 정리하는 것은 어려운 일입니다. 그래서 독서 경험을 효율적으로 관리하고 기록할 수 있는 독서노트 서비스를 만들었습니다.
  • 개발 기간 : 2023년 5월 20일 ~ 진행중
  • 최신배포 : 2023년 6월 28일

[2] 프로젝트 컨셉 & 주요 기능

프로젝트 컨셉

  • 기록은 간편하게 쓰고, 직관적으로 볼 수 있어야 한다고 생각합니다. 따라서 랜딩페이지 없이 접속을 하면 바로 게시글이 보여지게 했고, 헤더에 홈페이지 로고와 메뉴를 노출했습니다. 게시글 목록에서는 제목과 본문(200자), 카테고리, 작성일, 작성자의 닉네임이 노출됩니다.

  • 로그인/회원가입 페이지

    로그인을 누르거나 로그인 되지 않은 상태로 글쓰기 버튼을 누르면 로그인 페이지로 연결됩니다.
    독서와 글쓰기를 통해 스스로를 위한 온전한 시간을 갖기를 바라는 마음으로     Dive into your Ocean이라는 슬로건을 사용했습니다.

  • 계정이 없으신가요? 를 누르면 회원가입 페이지가, 비밀번호 찾기 를 누르면 비밀번호 재설정 이메일을 전송하는 페이지가 모달창으로 나타납니다. 버튼에 연결된 ModalOpen 함수에 매개변수로 string값을 주어서 각각 다른 컴포넌트를 포함하도록 작성했습니다.
  • Firebase를 이용한 Autu 기능
    • 회원가입 / 로그인 / 비밀번호 찾기
  • Glitch를 이용한 서버 호스팅
    • Glitch에 Json -server를 호스팅하여 DB 서버로 사용했습니다.
  • Firestore를 이용한 닉네임 설정
    • firebase에서 제공하는 데이터베이스인 Firestore를 사용하였습니다. 회원가입시 이메일,패스워드,닉네임을 기입하면 계정을 생성한 후 사용자의 uid 값을 문서id로 가지는 데이터 도큐먼트를 만들어 닉네임을 저장합니다.
  • 네이버 검색 API 를 이용한 책검색
    • 헤더 메뉴에 위치하여, 어느 페이지에서든 접근이 가능하며, 글을 쓰거나 다른사람의 글을 읽는 중에도 쉽게 검색할 수 있도록 모달 컴포넌트로 구현하였습니다.

[3] 배운점 & 아쉬웠던 점

  • 배운점
    • 프로젝트를 시작 할 때 저는 1차 목표로 CRUD를 완전하게 학습하고 구현하는 것으로 삼았습니다. 그리고 데이터를 성공적으로 다루게 된 다음에는 로그인 기능에 대해 고민하게 되었고, 세션,토큰을 이용한 인증에 대해 공부 한 후 firebase를 활용하여 사용자 인증을 구현했습니다.
    • 로그인/회원가입 기능을 추가한 다음에는 사용자 인증 정보를 컴포넌트 전역으로 관리하기 위해 Redux를 사용했습니다. 이런 과정을 통해 작은 프로젝트이지만 배포에 이르기까지 매일 꾸준히 한가지 이상의 과제를 해결하며 컴포넌트를 확장시켰습니다.
    • 리액트 라우터를 이용해 컴포넌트들의 레이아웃을 구성하는 법을 익히고, 데이터의 흐름과 생명주기에 대한 이해와 감각을 길렀습니다.
    • 뿐만 아니라 혼자 프로젝트를 진행하면서 일정 관리, 프로젝트 관리, 지속적으로 성잘 할 수 있는 공부법에 대한 인사이트를 얻었습니다. 자세한 내용은 블로그에 정리했습니다.
    • UI, 모든 것에는 이유가 있다.
      • 평소에 여러 웹사이트를 이용하면서도 당연하게 생각했던 것들, 예를 들어 작성완료 버튼의 위치, 화면의 비율, 어디로 리다이렉트 할것인지에 대해 직접 설계해봄으로써 모든 UI 요소는 디자인 뿐만 아니라 사용자의 편의와 효율성, 행동패턴을 고려하여 설계되어야 한다는 점을 알게 되었습니다.
  • 이런점이 아쉬웠습니다
  1. 현업에서의 개발 표준에 대해 배우고 싶습니다.
    • 비동기 작업을 처리하는 방법이나 미들웨어의 사용 등, 해결 과제가 생기면 학습 자료들을 통해 기능을 구현했지만 현업에서 실제로 사용하는 방법인지에 대한 궁금증이 있었습니다.
  1. 라우터를 더 똑똑하게 이용하고 싶습니다.
    • 게시글 목록, 게시글 상세보기 컴포넌트는 리액트 라우터의 loader를 사용해서 데이터를 공유하고 있습니다. 리덕스로 중앙집중적인 관리를 하기에 앞서, 이와 같이 라우터의 부모 컴포넌트-자식 컴포넌트의 관계를 잘 이용하면 작은 프로젝트에서는 더 컴팩트한 상태관리를 할 수 있지않을까 하는 생각을 했습니다.
